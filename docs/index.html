YEX - YAML Extended

A new, extended, and simpler version of YAML is on its way. YEX (YAML Extended) is designed for both newcomers and power users, with a new parser that supports features beyond the standard YAML specification. -w-

Key Features

Smarter Document Separation with `+page`

The **`+page`** command replaces the simple `---` separator with a full document system, allowing for built-in metadata like titles and descriptions.

+page  !llmsLazyload !noWrite
number: 0
name: YEX header
description: |
Official YEX/1.0 header...

Variables and Equations

Use a special variables page to store reusable data and define intelligent equations. Variables can be referenced anywhere, and equations are calculated by the parser.

global:
entities:
VecPaws:
$Epic Status$ = ($Epic Number$ * 10)
$Epic Number$ = 10

Variables are accessed with a clear scope. **`global`** variables are defined on a special page, while **`local`** variables are defined on a specific document.

# Access a global variable
$global.entities.VecPaws.Epic Status$

# Access a local variable
$local.entities.VecPaws.Epic Number$

Conditional Logic with `+if`

Add logic to your files! The **`+if`** and **`+ifNot`** statements allow for dynamic changes to the document based on conditions, turning YEX into a smart, self-modifying format. The parser understands a full range of logical and mathematical operators.

$Epic Percent$:
+if $Epic Number$ == 10,
set $Epic Percent$ = "100%"

Extensible RuleSets (`.yexr`) and Templates

YEX uses a new **`.yexr`** ruleset format to define features and provide a standard library of templates, ensuring compatibility and data integrity. Keys can be marked as required with an asterisk **`*`**.

+template:
  name: "LLM_WebLink"
  keys:
    title*:
      type: string
      required: true
    url*:
      type: string
      required: true
